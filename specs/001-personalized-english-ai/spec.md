# Feature Specification: 英语学习AI私人定制学习

**Feature Branch**: `001-personalized-english-ai`  
**Created**: 2026-03-06  
**Status**: Draft  
**Input**: User description: "我要开发一个 英语学习AI Web 应用，目标用户是 需要掌握英语的职场人员, 通过调用AI Api实现对用户的私人定制学习指导，杜绝市面上呆板千篇一律的英语单词学习软件。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 设定学习目标并生成计划 (Priority: P1)

作为需要提升英语能力的职场用户，我希望在首页快速选择学习领域与目标等级，并在短时间测试后获得个性化词汇学习计划。

**Why this priority**: 这是产品的核心价值入口，决定用户能否在首次使用时获得明确且可执行的学习路径。

**Independent Test**: 新用户从首页进入，完成领域/目标设置和短测，系统能生成首个学习计划；该流程可独立演示并交付 MVP 价值。

**Acceptance Scenarios**:

1. **Given** 用户首次进入首页, **When** 用户选择热门领域并选择目标等级, **Then** 系统展示该组合的学习目标说明并允许进入短测流程。
2. **Given** 用户已完成短测, **When** 用户提交测试结果, **Then** 系统在同一会话内生成与其水平匹配的首个词汇学习计划。
3. **Given** 用户有自定义领域需求, **When** 用户输入自定义学习领域并确认, **Then** 系统可基于该领域生成目标等级选项并进入计划生成流程。

---

### User Story 2 - 学习中动态调整计划 (Priority: P2)

作为持续学习的用户，我希望系统根据每次单词学习结果识别我的薄弱点，并动态调整后续学习内容。

**Why this priority**: 动态调整能力是“私人定制”体验的关键，直接决定长期学习效果与留存。

**Independent Test**: 用户完成至少两次学习后，系统能基于历史掌握情况自动调整后续计划难度和词汇重点。

**Acceptance Scenarios**:

1. **Given** 用户已存在学习计划, **When** 用户完成一次学习并提交掌握情况, **Then** 系统更新用户薄弱点并调整下一次学习内容。
2. **Given** 用户在同一薄弱点连续表现不佳, **When** 系统生成下一阶段计划, **Then** 系统增加该薄弱点相关词汇练习权重。

---

### User Story 3 - 课后作业与成就反馈 (Priority: P3)

作为完成一次学习的用户，我希望主动发起课堂作业测试，并获得可理解的掌握反馈与成就感。

**Why this priority**: 作业测试能强化记忆并形成正向反馈，提升学习完成率和持续动力。

**Independent Test**: 用户在完成单次学习后主动申请作业，系统生成对应测试并输出掌握结果与成长反馈。

**Acceptance Scenarios**:

1. **Given** 用户刚完成一次学习, **When** 用户点击“申请课堂作业”, **Then** 系统基于本次学习知识点生成测试题。
2. **Given** 用户完成课堂作业, **When** 用户提交答案, **Then** 系统返回掌握结果、薄弱点提示和下一步学习建议。

---

## Core Flow & Scope Guardrails *(mandatory)*

- 核心端到端流程：选择领域与目标等级 → 完成短测 → 生成个性化学习计划 → 完成一次学习并更新计划。
- 课后作业属于核心流程延伸，优先级低于首次计划生成与动态调整。
- 若排期冲突，必须先保证 P1 与 P2 完整闭环，再扩展增强体验。
- 以下内容明确为本期不做：社交功能、管理员后台、多语言支持。

## Usability Validation *(mandatory for impacted pages)*

受影响页面：首页、短测页、学习计划页、学习会话页、作业测试页。

- 初始加载：每个关键页面首屏可访问，核心信息在可视区域内可见。
- 主要点击交互：领域选择、目标等级选择、开始测试、提交学习结果、申请作业等操作可触发正确反馈。
- 表单提交：自定义领域输入、短测提交、作业提交均可成功提交并返回结果。
- 错误提示：网络失败、输入不完整、提交超时等常见错误必须给出明确可执行提示。
- 桌面端可用性：在桌面端主流视口下可完成核心流程，无关键控件遮挡或不可点问题。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在首页展示热门英语学习领域，并允许用户选择或输入自定义领域。
- **FR-002**: 系统 MUST 为每个学习领域提供多个目标等级（如看懂文章、能听能说、流畅交流、专家级别）。
- **FR-003**: 用户 MUST 能在短时间内完成英语词汇短测，用于评估当前水平。
- **FR-004**: 系统 MUST 基于用户领域、目标等级和短测结果生成首个个性化词汇学习计划。
- **FR-005**: 系统 MUST 在每次学习后记录用户词汇掌握情况并识别薄弱点。
- **FR-006**: 系统 MUST 根据最新掌握情况动态调整后续学习计划。
- **FR-007**: 用户 MUST 能在每次学习后主动申请课堂作业。
- **FR-008**: 系统 MUST 根据本次学习知识点生成课堂作业测试并在提交后给出结果反馈。
- **FR-009**: 系统 MUST 在页面刷新后保留关键学习数据（至少包括已选领域、目标等级、当前计划和最近学习进度）。
- **FR-010**: 系统 MUST 对常见错误场景提供明确提示，并给出下一步可操作指引。

### Non-Functional Requirements *(mandatory)*

- **NFR-001**: 用户从首次进入首页到完成核心操作（生成首个学习计划）平均耗时应不超过 3 分钟。
- **NFR-002**: 关键页面首屏必须可访问、可交互，并支持桌面端正常使用。
- **NFR-003**: 规格中的每条关键需求必须可通过验收场景或测试步骤验证。
- **NFR-004**: 体验文案应以清晰、可执行为原则，避免模糊提示。

### Key Entities *(include if feature involves data)*

- **User Learning Profile**: 用户学习画像，包含学习领域、目标等级、当前水平、阶段目标。
- **Vocabulary Assessment**: 词汇短测记录，包含测试题、用户答案、评估结果、时间戳。
- **Learning Plan**: 学习计划，包含阶段任务、词汇清单、难度分层、调整历史。
- **Learning Session Result**: 单次学习结果，包含掌握词汇、错误词汇、耗时、薄弱点标签。
- **Class Assignment**: 课堂作业，包含关联知识点、题目集合、作答结果、反馈结论。

## Edge Cases

- 用户未选择领域或目标等级即尝试开始短测时，系统应阻止继续并提示缺失项。
- 用户输入非常冷门或拼写异常的自定义领域时，系统应提示可用替代并允许继续。
- 短测中断或提交失败时，系统应保留已作答进度并允许重试。
- 学习结果上传失败时，系统应提示重试且不覆盖上一次已确认数据。
- 用户在短时间内重复申请课堂作业时，系统应防止重复生成冲突任务并给出说明。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 80% 以上新用户可在 3 分钟内完成“选择领域与目标 + 完成短测 + 获取首个学习计划”。
- **SC-002**: 95% 以上关键学习数据在页面刷新后可正确恢复。
- **SC-003**: 90% 以上常见错误场景可提供明确错误提示且用户知道下一步操作。
- **SC-004**: 桌面端用户在关键页面的核心操作成功率达到 95% 以上。
- **SC-005**: 完成至少 3 次学习的用户中，70% 以上可观察到后续计划内容发生与薄弱点相关的动态调整。

## Evidence Requirements *(mandatory)*

- 必须提供 lint、unit test、integration test 的通过证据。
- 必须对关键页面提供 Playwright smoke test 结果。
- 发生 UI 改动时，必须提供首页、学习计划页、学习会话页、作业测试页截图用于验收。
- 若存在自动修复流程，最多执行 3 轮，超过后必须转人工并记录失败原因。

## Assumptions & Dependencies

- 假设用户在使用前已具备基础登录态（本期不扩展管理员后台与多角色系统）。
- 依赖外部 AI 能力服务提供学习建议和题目生成能力；当服务异常时系统需提供降级提示。
- 热门学习领域列表可由业务配置定期更新，不要求本期支持复杂运营后台。
